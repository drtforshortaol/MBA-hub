Install MBA Hub

Windows / PC

Best method: install from Chrome.

  1. Open Chrome.
  2. Go to https://drtforshortaol.github.io/MBA-hub/.
  3. Look at the right side of the address bar.
  4. Click the Install app icon or the Open in app / Install icon.
  5. Choose Install.
  6. After it opens as an app window, right-click its icon on the taskbar.
  7. Click Pin to taskbar.

Windows if Install Icon Is Missing

  1. Click the Chrome three-dot menu.
  2. Choose Save and share.
  3. Choose Install page as app or Create shortcut.
  4. If available, check Open as window.
  5. Click Create or Install.

iPhone

  1. Open Safari.
  2. Go to https://drtforshortaol.github.io/MBA-hub/.
  3. Tap the Share button.
  4. Scroll down and tap Add to Home Screen.
  5. Rename it MBA Hub 2.2.2.
  6. Tap Add.

The Hub will appear like an app icon on your iPhone Home Screen.

Android

Best method: install from Chrome.

  1. Open Chrome on the Android phone or tablet.
  2. Go to https://drtforshortaol.github.io/MBA-hub/.
  3. Tap the three-dot menu in the upper-right corner.
  4. Tap Add to Home screen.
  5. If Chrome offers Install app, choose Install app.
  6. If Chrome offers Create shortcut, choose Create shortcut.
  7. Rename it MBA Hub 2.2.2.
  8. Tap Add or Install.

The Hub should now appear as an app icon on the Android Home Screen.

After Installing

Open the installed Hub icon and test:

  1. Install opens and closes.
  2. Help opens and closes.
  3. Clear Cache works.
  4. Information Center opens.
  5. Search works.
  6. Return to Hub works from inside individual apps.

If the Installed Version Looks Old

  1. Open Safari or Chrome directly.
  2. Go to https://drtforshortaol.github.io/MBA-hub/index.html?v=2.2.2.
  3. Tap or click Clear Cache.
  4. Remove the old Home Screen or desktop icon.
  5. Reinstall from the clean URL: https://drtforshortaol.github.io/MBA-hub/.

Help

Refresh Hub

  1. Tap Clear Cache.
  2. Let the Hub reload.
  3. Confirm the version at the bottom shows the newest date.

PC Chrome Hard Refresh

Press Ctrl + Shift + R to force Chrome to reload the newest Hub files.

iPhone Safari Cache

  1. Open iPhone Settings.
  2. Go to Apps → Safari → Advanced → Website Data.
  3. Search for the site, delete matching website data, then reopen Hub.

Android Chrome Cache

  1. Open Chrome.
  2. Tap the three-dot menu.
  3. Go to Settings → Privacy and security.
  4. Tap Clear browsing data.
  5. Select cached images/files and site data if needed.
  6. Reopen the Hub from the clean URL.

Broken App Link

If an app opens incorrectly or Return to Hub gives 404, check that the app folder is inside apps/app-folder-name/ and that the app uses ../../index.html for the MBA Hub button.

App Still Shows Old Version

Use Clear Cache, then reload. If the old version still appears, close and reopen the browser.

On iPhone or Android, remove the old Home Screen icon if needed and add the Hub again after clearing website data.

Service Worker Blocking Updates

If the live Hub does not match the files you uploaded, the browser may be showing old cached files.

On PC Chrome, open Developer Tools, go to Application → Service Workers, unregister the old worker, then clear site data and reload.

Tags and Cross-Links

Tags should appear on each app card. Clicking a tag should filter the Hub search to that tag.

Related app links can come from two places: relatedApps in hub-registry.js, or automatic shared-tag matching between apps.

Intentional special cross-links can be stored in hub-links.js.

References and Notes

Hub-wide app listings are maintained in hub-registry.js. Hub-wide tag definitions are maintained in hub-tags.js. Optional intentional cross-links are maintained in hub-links.js.

Individual app content should remain inside each app folder. Individual app data.js files are for that app’s own internal content, not for the root Hub registry.

Hub Registry Safety

Root Hub registry files use unique names: hub-registry.js, hub-tags.js, and hub-links.js.

The root Hub should not use data.js as the long-term registry.
